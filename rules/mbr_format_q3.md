# Q3: #OneTeam Initiatives Format Rules

**Part of**: MBR Generation Specification v3.0
**See also**: [Q3 Examples](../examples/mbr_examples_q3.md)

## Format Type

**SIFT-Sourced Initiatives** - Cross-account technical patterns with documented SA engagement

## Critical Constraint

**ONLY include initiatives that**:
- Are documented in SIFT insights
- Show cross-account technical patterns
- Have documented SA engagement
- Have measured pipeline impact

**If no SIFT-sourced initiatives found**: Use placeholder with guidance

## Entry Format

```
[Initiative Title] (SIFT: [creator alias], [date]): [Description of
workshops/programs delivered] with [quantified impact]. [Specific accounts]
launched [outcomes]. [SA deliverable published]. [Next: scale plans].
```

## Required Elements

### Must Include
- **SIFT source citation** (creator alias and date)
- **Type of initiative**: Workshop, GameDay, Immersion Day, TFC campaign, reference architecture
- **Number of accounts** engaged
- **Pipeline impact** ($ amount or account count)
- **Specific customer names** that participated
- **SA deliverable** (reference architecture, TFC post, documentation)
- **Next steps** for scaling (WWSO, GTM codes)

### Initiative Types
- Workshops delivered (hands-on technical training)
- Reference architectures built (reusable technical patterns)
- TFC campaigns (Tech Field Community content)
- GameDays (hands-on labs and exercises)
- Immersion Days (multi-day deep dives)
- Field programs (structured enablement)

## Critical Rules

### DO:
- Cite SIFT source with creator and date
- Quantify pipeline impact ($ or account count)
- Name specific accounts that participated
- Document SA deliverables produced
- Include scaling plans with codes (TFC-####, GTM-####)

### DO NOT:
- Include account-level observations (those go in Q1 or Q4)
- Assume or fabricate initiatives without SIFT evidence
- Include future plans without documented execution
- List initiatives without SA engagement proof
- Include initiatives without cross-account patterns

## Content Scope

**Belongs in Q3**:
- Multi-account workshops with documented attendees
- Reference architectures used by 3+ accounts
- TFC posts that generated pipeline activity
- GameDays with measured outcomes
- Cross-group technical programs with SA ownership

**Does NOT belong in Q3**:
- Single account technical wins (Q4 Highlights)
- Account-level observations (Q1 or Q4)
- Planned initiatives not yet executed
- Observations without SA engagement
- Activities without cross-account patterns

## Placeholder Guidance

If no SIFT-sourced initiatives found, use:

```
**#OneTeam Initiatives**: No cross-account technical initiatives found in
SIFT from the last 30 days. Group managers should submit SIFT insights for:
- Multi-account workshops delivered (with attendee list)
- Reference architectures published (with TFC codes)
- GameDays or Immersion Days (with pipeline impact)
- Field programs with documented SA engagement

Contact your SA leadership to document cross-account technical patterns.
```

## Quality Checks

Before considering Q3 complete, verify:
- [ ] Every initiative cited with SIFT source
- [ ] Pipeline impact quantified ($ or accounts)
- [ ] Specific customer names included
- [ ] SA deliverable documented
- [ ] Scaling plans with codes (TFC/GTM)
- [ ] NO account-level observations included
- [ ] All initiatives show cross-account patterns
- [ ] OR placeholder used if none found
