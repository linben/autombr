# Phase 3: Writing

**Part of**: MBR Generation Specification v3.0
**See also**: Format rules for each quadrant

## Goal

Generate narrative following style rules. This is where actual content creation happens.

## Writing Steps

### Step 3.1: Write Q1 (Health of Business)

See detailed format rules: [Q1 Format Rules](mbr_format_q1.md)

**Key Requirements**:
- Write as single flowing paragraph (NOT bullets)
- Include all business metrics with specific numbers
- Follow with narrative observation paragraphs (3-5 lines each)
- Each observation: pattern → accounts → $ ARR → [Ask]

### Step 3.2: Write Q2 (Risks/Blockers)

See detailed format rules: [Q2 Format Rules](mbr_format_q2.md)

**Key Requirements**:
- Format: [TAG] Bold Title. Inline prose. [Next Steps] at end
- NO STAR labels (Situation:/Task:/Action:/Result:)
- Priority order: [R] → [B] → [C] → [O]
- 4-6 entries maximum

### Step 3.3: Write Q3 (#OneTeam)

See detailed format rules: [Q3 Format Rules](mbr_format_q3.md)

**Key Requirements**:
- ONLY SIFT-sourced cross-account technical patterns
- Must document SA engagement and pipeline impact
- If none found, use placeholder with guidance

### Step 3.4: Write Q4 (Highlights/Lowlights)

See detailed format rules: [Q4 Format Rules](mbr_format_q4.md)

**Key Requirements**:
- Format: [H]/[L] Bold Title. 3-5 sentence mini-story. [Next Steps]
- NO STAR labels, pure narrative
- Include competitive context within story
- Highlights: 5-8 entries | Lowlights: 3-5 entries

### Step 3.5: Generate Appendices

- Appendix F: Top 20 Opportunities table (must have 20 entries)
- Appendix G: Territory Group Summary (all 5 groups)
- Charts: YTD usage, GenAI adoption, pipeline vs spend

See: [Document Structure](mbr_document_structure.md) for appendix details

## Output

Complete MBR draft in 2x2 format ready for validation.
