# Phase 2: Content Planning

**Part of**: MBR Generation Specification v3.0
**See also**: [CLAUDE.md](../CLAUDE.md) for project setup

## Goal

Outline content structure before writing. Create a content plan that can be reviewed.

## Planning Steps

### Step 2.1: Plan Q1 (Health of Business)
- [ ] Draft key metrics for opening paragraph
- [ ] Select 3-5 top observations with accounts named
- [ ] Ensure no generic preambles, only new content
- [ ] Verify all groups represented

### Step 2.2: Plan Q2 (Risks/Blockers)
- [ ] Select 4-6 most critical items
- [ ] Prioritize: [R] → [B] → [C] → [O]
- [ ] Group systemic issues (don't repeat similar risks)
- [ ] Assign owner and next step date for each

### Step 2.3: Plan Q3 (#OneTeam)
- [ ] Filter SIFT-sourced initiatives only
- [ ] Verify SA engagement and pipeline impact documented
- [ ] Exclude account-level observations (those go in Q1/Q4)
- [ ] Use placeholder if no verified initiatives found

### Step 2.4: Plan Q4 (Highlights/Lowlights)
- [ ] Select 5-8 highlights with business impact
- [ ] Select 3-5 lowlights with lessons learned
- [ ] Ensure each has mini-story arc (what/why/next)
- [ ] Include competitive context where relevant

## Output

Content outline with item selection checklist showing:
- Which accounts/insights selected for each quadrant
- Priority order for risks/blockers
- Verification that all groups are represented
- Approval checkpoint before writing begins
